Swift: Generics

Generics allow you to create reusable code that can work with different data types. By using generics, you can write more flexible and type-safe code without compromising on type-safety.




Understanding Generics

A generic type is a placeholder for a specific type that will be determined at runtime. This allows you to create functions, classes, and structs that can work with different data types without sacrificing type safety. Think of it as: "I don't know what the type is yet!".

func swap(_ a: inout T, _ b: inout T) {
    let temp = a
    a = b
    b = temp
}

var a = "A"
var b = "B"
swap(&a, &b)
print(a) //B
print(b) //A


In the above example, the `swap` function is generic. It takes two inout parameters of type `T` and swaps their values. The `T` can be any type, such as `Int`, `String`, or a custom struct. A feature of generics is that it forces both inputs to be of the same type. if for example `b` was of type int while `a` is a string, a compile time error would be thrown.

Generic Functions

Generic functions are functions that can work with different data types. They are defined using angle brackets (`<>`) after the function name, followed by a placeholder type.

func findMax(_ array: [T]) -> T? {
    guard !array.isEmpty else {
        return nil
    }
    var max = array[0]
    for element in array {
        if element > max {
            max = element
        }
    }
    return max
}

print(findMax([3,4,5,6,2,3,4,5,6,7,3])) // 7
print(findMax(["Hello", "Swift"])) // Swift


In this example, the `findMax` function takes an array of elements of type `T` and returns the maximum element. The `T` must conform to the `Comparable` protocol, which means it must implement the `==` and `<` operators. We were able to use it successfully on both arrays of integers and string, but NOT a mix of the two.




Generic Structs and Classes

You can also create generic structs and classes. This allows you to create reusable data structures that can work with different data types.

struct Stack {
    private var items = [T]()

    mutating func push(_ item: T) {
        items.append(item)
    }

    mutating func pop() -> T? {
        return items.popLast()
    }
}

var stack = Stack()
stack.push(3)
stack.push(2)
stack.push(1)

while let ii = stack.pop() {
    print(ii)
}


In this example, the `Stack` struct is generic. It can store elements of any type `Element`. Notice how we internally used and Array to store the elements which uses the same generic type as we declared with the Stack.

Benefits of Generics

• Reusability: Generics allow you to write reusable code that can work with different data types, without sacrificing type safety.
• Type Safety: Generics help you write type-safe code, which can prevent runtime errors.

Best Practices

• Use meaningful type names: Choose type names that clearly convey the purpose of the generic type.
• Consider using associated types: Associated types can make your generic types more flexible.



Generics are a powerful feature in Swift that can help you write more reusable, type-safe, and flexible code. By understanding how to use generics effectively, you can create more robust and maintainable applications.
