Swift: Subscripts

Subscripts are a powerful feature in Swift that allow you to access elements in collections concisely, think of how you access elements in an array or a dictionary. By defining custom subscripts, you can create more expressive and intuitive collection types that closely resemble how arrays are accessed.




Understanding Subscripts

A subscript is a computed property that takes an index as a parameter and returns a value of a specific type, that's it really. It's a function that trades in some sort of key for some value:

struct MyCollection {
    var items = [Int]()

    subscript(index: Int) -> Int? {
        get {
            if index >= 0 && index < items.count {
                return items[index]
            } else {
                return nil
            }
        }
        set {
            if let newValue = newValue {
                if index >= 0 && index < items.count {
                    items[index] = newValue
                }
            }
        }
    }
}

let initialArray = [1,2,3,4,5,6,7]
let collection = MyCollection(items: initialArray)
print(collection[5])


In the above example, the `MyCollection` struct defines a subscript that allows you to access and modify elements in the `items` array using an index.

However, you dont always need to have some sort of built in collection to backup your custom ones, we can be creative/useless with something like the following:
struct MyCollection {
    subscript(key: Any) -> String {
        get { "\(key)" }
    }
}
let collection = MyCollection()
print(collection[5])
print(collection["Hello, Swift!"])
print(collection[[1,2,3,4]])





Key Features of Subscripts

• Customizable: You can define subscripts with any number of parameters and return types, even variadic parameters.
• Read-only or read-write: Subscripts can be read-only (only `get`) or read-write (both `get` and `set`).
• Type safety: Subscripts provide type safety by specifying the types of the index and the returned value, additionally, you can specify a generic for your custom collections.
• Flexibility: Subscripts can be used to implement various collection behaviors, such as accessing elements by index, key or even ranges.

Common Use Cases

• Matrix operations: Implement matrix operations using subscripts to access elements by row and column.
• Sparse data structures: Store sparse data efficiently using subscripts.
• Specialized data structures: Create specialized data structures with custom access patterns.

Best Practices

• Use clear and descriptive names: Choose names for your subscripts that clearly convey their purpose.
• Validate input: Ensure that the input to your subscripts is valid to avoid errors.
• Consider performance implications: Subscripts can have performance implications, especially when dealing with large collections.
• Use read-only subscripts when appropriate: If your collection should be read-only, define read-only subscripts.

Advanced Topics

• Multiple subscripts: You can define multiple subscripts with different parameters and return types.
• Subscripts with ranges: Implement subscripts that take a range as an index.
• Subscripts with custom types: Create subscripts that take custom types as indices.

struct MyCollection {

    subscript(key: String) -> String {
        get { key.uppercased() }
    }

    subscript(range: Range) -> [Int] {
        get { range.map({$0 + 1}) }
    }

}

let collection = MyCollection()
print(collection["StringKey"])
print(collection[0..<5])




Subscripts are a powerful feature in Swift that allow you to create custom collection types with expressive and intuitive access patterns which resembles what most coders are already used to. Remember to choose the right use cases for subscripts and make sure it won't unintentionally introduce complexity and redundancy instead.
