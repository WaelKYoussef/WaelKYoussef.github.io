Swift: Collection Types

Collections are fundamental data structures in Swift that allow you to store and manage groups of values. Understanding and effectively using collection types is essential for structuring you data, and you'll come to know that mastering structuring your data is key to many amazing feats. So let's look at the various collection types available in Swift, their key features, and best practices for using them.



Arrays

Arrays are ordered collections of values of the same type, known as lists in some other languages. They're ideal for storing sequences of elements, such as a list of names or a series of numbers.

let names = ["Alice", "Bob", "Charlie"]
let numbers = [1, 2, 3, 4, 5]


Key features:

• Ordered: Elements in an array are stored in a specific order.
• Mutable or immutable: You can create mutable arrays that can be modified or immutable arrays that cannot be changed.
• Indexing: Access elements in an array using their index (starting from 0).
• Fast access: You can access any element of an array using its index in constant time.

Dictionaries

Dictionaries, also known as Maps or HashMaps in other languages, are unordered collections of key-value pairs. They're useful for storing data that needs to be accessed by a key, such as a mapping of names to ages or a configuration file.

let person = ["name": "Alice", "age": 30]
let cities = ["Egypt": "Cairo", "USA": "New York"]


Keys can actually be anything, even instances. The only requirement for a class to be a key is to conform to the `Hashable` protocol:

class MyClass: Hashable {
    static func == (lhs: MyClass, rhs: MyClass) -> Bool {
        lhs.id == rhs.id
    }

    let id: String

    init(id: String) {
        self.id = id
    }

    func hash(into hasher: inout Hasher) {
        hasher.combine(id)
    }
}

let dict: [MyClass:String] = [
    MyClass(id: "111"): "One",
    MyClass(id: "222"): "Two",
    MyClass(id: "333"): "Three",
]

print(dict[MyClass(id: "222"), default: "Not found"]) // Prints "Two"


Key features:

• Unordered: Elements in a dictionary are not stored in a specific order.
• Key-value pairs: Each element in a dictionary consists of a key and a value.
• Unique keys: Keys in a dictionary must be unique.
• Flexible keys: Keys can be even class instances.



Sets

Sets are unordered collections of unique values of the same type. They're useful for storing a list of distinct elements, such as a set of unique numbers or a set of allowed characters.

let vowels = Set(["a", "e", "i", "o", "u"])
let numbers = Set([1, 2, 3, 3, 4])


Same as with dictionaries, all a set requires for an object to be in a set is to conform to `Hashable`:

class MyClass: Hashable {
    static func == (lhs: MyClass, rhs: MyClass) -> Bool {
        lhs.id == rhs.id
    }

    let id: String

    init(id: String) {
        self.id = id
    }

    func hash(into hasher: inout Hasher) {
        hasher.combine(id)
    }
}

let set: Set = Set([
    MyClass(id: "111"),
    MyClass(id: "111"),
    MyClass(id: "111"),
    MyClass(id: "222"),
    MyClass(id: "222"),
    MyClass(id: "333"),
    MyClass(id: "333"),
    MyClass(id: "333"),
    MyClass(id: "333"),
    MyClass(id: "333"),
    MyClass(id: "333")
])

print(set.count) // Prints 3


Key features:

• Unordered: Elements in a set are not stored in a specific order.
• Unique values: Sets cannot contain duplicate values.
• Membership testing: Efficiently check if an element is contained in a set -constant time-.

Choosing the Right Collection Type

The choice of collection type depends on the specific requirements of your application. Here are some guidelines:

• Arrays: Use arrays for ordered sequences of elements like a list of items.
• Dictionaries: Use dictionaries for key-value pairs where you need to access elements by a key like mapping class instances to server identifiers.
• Sets: Use sets for unordered collections of unique values and existence-checking.



Additional collections features

• Subscripting: Access elements in a collection using subscripts i.e. the *object[] syntax.
 Iteration: Iterate over elements in a collection using loops.
• Mutable and immutable variants: Create mutable or static immutable collections.
• Performance optimizations: Swift's collection types are optimized for performance base on the use case.

Best Practices

• Choose the appropriate collection type: Select the collection type that best suits your needs. Just because one type can be used doesn't mean that it's the best choice.
• Use clear and descriptive names: Give your collections meaningful names to improve readability, true for everything else.
• Avoid unnecessary copying: Minimize copying of collections to improve performance.



Collection types are essential building blocks in Swift programming. By understanding the different collection types and their features, you can unlock a superpower of performance and optimization. Remember to choose the right collection type for your specific use case and follow best practices to ensure your code is maintainable and performant.
